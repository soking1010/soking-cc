企業內訓

UX 講師：王彥博 / Soking

Soking 是一位資深軟體產品經理，也是一位 UX 講師。曾在跨國影音平台、新創公司、網路媒體等數位產品領域工作超過十五年，擁有豐富的數位產品開發與營運的經驗。

Soking 擅長以 UX 設計工作坊帶領學員實際操作，協助學員更深入的了解 UX 設計。 線上課程《產品分析入門：UX 設計師的思考術》在 Hahow 線上課程平台上受到了廣泛的好評，已有超過千人購買，平均評價為 4.7 星。


教學領域

  • UXD 產品設計、產品經理技能
  • 數位軟體規劃、資訊架構設計
  • 用戶旅程、需求訪談、消費者市場需求研究

企業內訓案例

玉山銀行 / 金融業

透過四天課程協助商業分析師（BA），培訓 UXD 產品設計的專業素養。

街口支付 / 數位金融

透過半日課程，培訓內部產品經理關於用戶滿意度問卷設計的能力。

iChef 資廚 / 餐飲科技

透過整日需求訪談工作坊，提供產品開發團隊需求同理的溝通練習。

皇喬投資集團 / 資訊軟體業

透過一日課程培訓用戶旅程與易用性的 UX 思維。

 

企業內訓課程項目

目前常態開設之企業內訓課程，皆以資訊軟體從業人員（如：產品經理、產品設計師、UIUX 設計師、商業分析師、工程師等）為培訓目標之授課內容。 如需客製化調整授課主題，歡迎來信洽詢： service@soking.cc

  • 用戶訪談工作坊（7 hr）
  • 需求訪談工作坊（7 hr）
  • 資訊架構設計工作坊（7 hr）
  • 用戶體驗優化工作坊（4 hr）
  • 目標客群與用戶旅程工作坊（4 hr）
  • 滿意度問卷設計工作坊（4 hr）

 

用戶訪談工作坊（7 hr）

目的：根據商業模式設定目標族群，進行用戶需求的研究，用以提供產品策略的參考依據。 

課程介紹

  •  了解用戶訪談的基本知識以及應用情境
  • 練習用戶訪談技巧與現場流程
  • 練習訪綱設計
  • 練習將訪談成果如何轉換為需求洞察

 

 

需求訪談工作坊（7hr）

目的：練習辨識需求的情境與過程，幫助團隊提昇需求溝通的品質。

課程介紹

  • 了解如何辨識需求的類別
  • 練習與利害關係人溝通的核心技巧
  • 練習快速認識陌生產業的提案前準備方法

 

資訊架構設計工作坊（7hr）

目的：練習建立領域事件的內容模型，用以更精準的溝通軟體規格需求。

課程介紹

  • 服務流程設計與任務拆解
  • 內容模型與有效訊息的分析法
  • 認識領域事件分析，用以解構資訊內容流向

 

用戶體驗優化工作坊（4hr）

目的：認識產品服務的用戶體驗如何進行分析評估，提出改善計畫。

課程介紹

  • 可使用企業指定案例作為教學對象
  • 現場執行用戶體驗優化的測試與觀察
  • 透過易用性原則分類問題優先級
  • 改善方針提案發想

 

 

目標客群與用戶旅程工作坊（4hr）

目的：認識如何根據市場競爭狀態，辨認適合的目標客群。

課程介紹

  • 用戶體驗分析矩陣
  • 目標客群的商業分析框架
  • 用戶旅程與接觸點

 

滿意度問卷設計工作坊（4hr）

目的：認識測量用戶滿意度的問卷設計原理，用以評估產品服務的目標客群觀點。

課程介紹

  • NPS 滿意度問卷
  • 狩野分析需求評估問卷
  • 游擊訪談問卷與行為編碼分析

 

 

企業內訓詢價流程

  • 請來信至 service@soking.cc，簡述貴組織本次的培訓目標。
  • 可以預約一次線上會議諮詢，與相關主管或需求單位討論培訓主題的需求。
  • 提供報價單與課程大綱，回簽確認可執行。
  • 約定授課日期與上課人數。

注意事項

  • 企業內訓的請款發票，將在開課當週以電子發票開立，並以 Email 寄送。
  • 課前需確認的事項為：
    • 上課場地的確切地址，講師很容易迷路。
    • 當日上課的起始時間，我方將提前半小時進場佈置。
    • 參與課程人數，將會準備當日講義、課程演練用文具。
    • 上課場地需要可投影設備，訊號源可以 HDMI 輸出。
  • 課程當日可以拍攝或錄影，以供邀課單位內部使用。
  • 課後將提供授課當日的上課簡報，以供邀課單位內部複習使用。
  • 講師方的助教會拍攝當日授課狀況，並使用於後續社群曝光與客戶案例用途。
